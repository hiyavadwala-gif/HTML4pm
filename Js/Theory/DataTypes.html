• There are 2 type of DataTypes
  • *Primitive Data Types- it showcase a single value ,stored in direct in memory *Non-Primitive DataTypes- Represent collections or complex entities. Stored by reference.
  • Primitive Data Types-
  • Type Description Example Value (Conceptually) String Sequence of characters used for text. "Hello", 'A' Number Represents numeric values (integers, decimals). 10, 3.14
  • Boolean Represents truth values. true, false Undefined A variable declared but not initialized.
    • undefined
  • Null Represents intentional absence of value. null
  • BigInt Used for integers larger than Number can handle. 9007199254740991n
  • Symbol Represents unique and immutable identifiers. Symbol('id')
  • ● Strings are enclosed in single, double, or backtick quotes.
  • ● Number includes both integers and floating-point numbers.
  • ● Boolean helps in decision-making (true/false).
  • ● Undefined means a variable is declared but not assigned.
  • ● Null is an intentional empty value.
  • ● BigInt is used for very large numbers beyond the safe integer limit.
  • ● Symbol is mainly used as object property keys to ensure uniqueness.
  • Non-Primitive DataTypes
  • Object A collection of key–value pairs. Used to represent structured data. .
  • Array An ordered list of values (a type of object).
  • Function A block of reusable code that can be executed
  • Feature Primitive Non-Primitive Storage Stored directly in the stack memory. Stored in the heap; reference stored in stack. Mutability Immutable (cannot be changed). Mutable (can be modified). Copy Behavior Copy creates a new value. Copy creates a new reference to the same object. Examples String, Number, Boolean Object, Array, Function
